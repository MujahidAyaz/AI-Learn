Load the Data
Plain English We load a real dataset of 768 patients. Each row is one patient. The last column Outcome is the answer — 1 means they got diabetes, 0 means they didn't. Everything else is the clues the model will learn from.
python 
# Load the diabetes dataset from a URL
url = "https://raw.githubusercontent.com/plotly/datasets/master/diabetes.csv"

Step 2 of 7
3
Split Features & Label, Then Train/Test Split
Plain English We separate the clues (X) from the answer (y). Then we split everything into two piles — 80% for training (the model studies this), 20% for testing (hidden until the final exam). The model never sees the test set while learning.
python 
# X = the clues (all columns except the last one)
X = df.drop("Outcome", axis=1)

# y = the answer we want to predict (0 or 1)
y = df["Outcome"]

# Split: 80% train, 20% test
# random_state=42 just means the split is always the same (reproducible)
X_train, X_test, y_train, y_test = train_test_split(
    X, y,
    test_size=0.2,       # 20% goes to test
    random_state=42     # so results are reproducible
)

print(f"Training patients: {len(X_train)}")  # → 614
print(f"Testing patients:  {len(X_test)}")   # → 154
Output
Training patients: 614
Testing patients: 154
train/test split X (features) y (label)
Step 3 of 7
4
Scale the Features
Plain English Age goes from 20–80. Glucose goes from 0–200. Blood pressure is different again. If we leave them as-is, the model thinks glucose matters 10× more than age just because the numbers are bigger — that's unfair. Scaling puts all features on the same playing field (roughly 0 to 1).
python 
# StandardScaler rescales features so they're all on same scale
scaler = StandardScaler()

# fit_transform = learn the scale FROM training data, then apply it
X_train_scaled = scaler.fit_transform(X_train)

# transform only = apply the SAME scale to test data
# (never fit on test data — that would be cheating!)
X_test_scaled = scaler.transform(X_test)

# Before scaling: Glucose might be 148
# After scaling:  Glucose becomes something like 0.86
scaling StandardScaler normalization
Step 4 of 7
5
Train the Model
Plain English This is the "studying" phase. We hand the model the training data and let it run thousands of loops — making guesses, measuring the loss (how wrong it was), and using gradients to nudge its weights in the right direction. The C parameter controls regularization — how much we penalize the model for overcomplicating things.
python 
# Create the model
# C=1.0 → regularization strength (smaller = more regularization)
# max_iter → how many loops to run while learning
model = LogisticRegression(C=1.0, max_iter=1000)

# .fit() = "study this data and learn from it"
# This is where loss function + gradient descent + regularization all happen
# sklearn handles all the math for you under the hood
model.fit(X_train_scaled, y_train)

print("Training complete!")
print("Learned weights:", model.coef_)  # how much each feature matters
Output
Training complete!
Learned weights: [[ 0.31 1.18 -0.22 0.07 0.14 0.69 0.34 0.38]]
← Glucose (1.18) matters most. Blood pressure (-0.22) slightly lowers risk.
loss function gradient descent regularization (C) weights
Step 5 of 7
6
Test the Model & Check Accuracy
Plain English Final exam time. We show the model the 154 patients it has never seen. It makes predictions. We compare to reality and see how well it did. Accuracy tells us overall correctness, but precision and recall tell us about specific types of mistakes — which matters more in medical diagnosis.
python 
# Make predictions on the test set (never seen before)
y_pred = model.predict(X_test_scaled)

# Also get probabilities (how confident is the model?)
y_proba = model.predict_proba(X_test_scaled)

# Overall accuracy: what % did it get right?
acc = accuracy_score(y_test, y_pred)
print(f"Accuracy: {acc * 100:.1f}%")

# Detailed breakdown: precision, recall, f1
print(classification_report(y_test, y_pred))
Output

Step 6 of 7
7
Predict a New Patient
Plain English This is the whole point — using the trained model on real new data. A new patient walks in. We plug their info in and get a probability and a prediction instantly.
python 
# A brand new patient — model has never seen this person
# [Pregnancies, Glucose, BloodPressure, SkinThickness,
#  Insulin, BMI, DiabetesPedigreeFunction, Age]
new_patient = [[2, 138, 72, 30, 140, 33.6, 0.627, 45]]

# Scale using the SAME scaler we trained on (important!)
new_patient_scaled = scaler.transform(new_patient)

# Predict: 0 = no diabetes, 1 = diabetes
prediction = model.predict(new_patient_scaled)

# Get the probability
probability = model.predict_proba(new_patient_scaled)[0][1]

print(f"Prediction:  {'Diabetes' if prediction[0] == 1 else 'No Diabetes'}")
print(f"Confidence:  {probability * 100:.1f}% chance of diabetes")
Output
Prediction: Diabetes
Confidence: 73.2% chance of diabetes
What You Just Built

Train/Test Split

80% to learn, 20% for honest evaluation — model never cheats on the exam

Loss Function

Cross-entropy — measures how wrong each guess is and gives a penalty score

Gradient Descent

Runs 1000 loops, nudging weights downhill toward lower loss each time

Regularization (C)

Stops model from memorizing noise — forces it to find real patterns

Scaling

Puts all features on equal footing so no feature dominates unfairly

Accuracy + Recall

78.6% accurate overall — recall tells if it catches real diabetes cases
