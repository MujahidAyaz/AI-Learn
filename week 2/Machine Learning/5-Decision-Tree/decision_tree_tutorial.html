3
Train an Unpruned Tree (Overfit on Purpose)
Plain English First, we let the tree grow with NO limits. It will ask questions until every single patient is perfectly classified. This is overfitting — it memorizes the training data. We do this on purpose so you can see the problem before we fix it.
python
# No limits = tree grows until every patient is perfectly classified
# This will overfit — memorize training data
tree_overfit = DecisionTreeClassifier(random_state=42)
tree_overfit.fit(X_train, y_train)

# Check training accuracy vs test accuracy
train_acc = accuracy_score(y_train, tree_overfit.predict(X_train))
test_acc  = accuracy_score(y_test,  tree_overfit.predict(X_test))

print(f"Overfit tree depth:    {tree_overfit.get_depth()}")
print(f"Training accuracy:     {train_acc * 100:.1f}%")
print(f"Test accuracy:         {test_acc  * 100:.1f}%")
print()
print("Gap = the tree memorized instead of learning!")
Output
Overfit tree depth: 17
Training accuracy: 100.0% ← memorized everything
Test accuracy: 69.5% ← fails on new patients

Gap = the tree memorized instead of learning!
This is the overfitting problem 100% on training, 69.5% on test. The tree is 17 levels deep asking ridiculous questions like "is glucose exactly 148 AND age exactly 45?" — that matches ONE patient, not a real pattern.
overfitting depth 17 memorization
Step 3 of 8
4
Prune the Tree — Fix Overfitting
Plain English Now we limit how deep the tree can go. max_depth=4 means it can only ask 4 questions before giving an answer. This forces it to find general patterns that work on new patients — not memorize the training ones.
python
# max_depth=4  → only 4 questions deep (pruned)
# min_samples_split=10 → only split a group if it has 10+ patients
# min_samples_leaf=5   → each final answer must cover 5+ patients
tree = DecisionTreeClassifier(
    max_depth=4,
    min_samples_split=10,
    min_samples_leaf=5,
    criterion="gini",      # use Gini impurity to pick best questions
    random_state=42
)

tree.fit(X_train, y_train)

# Compare again
train_acc = accuracy_score(y_train, tree.predict(X_train))
test_acc  = accuracy_score(y_test,  tree.predict(X_test))

print(f"Pruned tree depth:   {tree.get_depth()}")
print(f"Training accuracy:   {train_acc * 100:.1f}%")
print(f"Test accuracy:       {test_acc  * 100:.1f}%")
Output
Pruned tree depth: 4
Training accuracy: 80.3% ← not perfect, but that's fine
Test accuracy: 77.3% ← much closer! generalizes well
Why lower training accuracy is actually good here The gap between train (80%) and test (77%) is tiny — the tree learned real patterns, not noise. Before, the gap was 100% vs 69% — terrible. A small gap means the model generalizes well.
max_depth pruning generalization gini impurity

Step 5 of 8

Visualize the Tree
Plain English We can draw the tree as an actual diagram. Each box shows the question asked, how many patients are in that group, and how pure the group is. This is one of the biggest advantages of decision trees over logistic regression — you can show this to a doctor and they'll understand it.
python
# Draw the decision tree as a visual diagram
fig, ax = plt.subplots(figsize=(20, 8))

plot_tree(
    tree,
    feature_names=feature_names,          # show feature names not numbers
    class_names=["No Diabetes", "Diabetes"],  # show class names
    filled=True,                          # color boxes by majority class
    rounded=True,                         # rounded corners
    fontsize=10,
    ax=ax
)

ax.set_title("Decision Tree — Diabetes Prediction (max_depth=4)",
             fontsize=14, fontweight="bold")

plt.tight_layout()
plt.savefig("decision_tree_diagram.png", dpi=150, bbox_inches="tight")
plt.show()

# Orange boxes → majority are diabetic
# Blue boxes   → majority are non-diabetic
# Darker color → purer group (more confident answer)
How to read the colored boxes Orange = mostly diabetes. Blue = mostly no diabetes. The darker the color, the purer the group — meaning the model is more confident in that box's answer.

Evaluate & Feature Importance
Plain English After testing, we check accuracy like before. But decision trees give us a bonus — feature importance. It tells you exactly which features the tree relied on most when making its decisions. Glucose almost always wins.
python
# Predictions on the hidden test set
y_pred  = tree.predict(X_test)
y_proba = tree.predict_proba(X_test)[:, 1]

print(f"Accuracy : {accuracy_score(y_test, y_pred) * 100:.1f}%")
print(f"ROC-AUC  : {roc_auc_score(y_test, y_proba):.3f}")
print()
print(classification_report(y_test, y_pred,
      target_names=["No Diabetes", "Diabetes"]))

# Feature importance — which clues did the tree use most?
importances = tree.feature_importances_
print("Feature Importance (how much each clue was used):")
for name, imp in sorted(zip(feature_names, importances),
                          key=lambda x: x[1], reverse=True):
    bar = "█" * int(imp * 40)
    print(f"  {name:<30} {imp:.3f}  {bar}")

Predict a New Patient
Plain English Same as logistic regression — plug in a new patient, get an instant prediction. The difference: you can trace exactly which questions the tree asked to reach that answer. Full transparency.
python
# New patient — model has never seen them
new_patient = pd.DataFrame([[2, 138, 72, 30, 140, 33.6, 0.627, 45]],
                              columns=feature_names)

# No scaling needed for trees!
prediction  = tree.predict(new_patient)[0]
probability = tree.predict_proba(new_patient)[0][1]

print(f"Prediction:  {'⚠ Diabetes Likely' if prediction == 1 else '✓ No Diabetes'}")
print(f"Confidence:  {probability * 100:.1f}% probability of diabetes")
print()

# You can trace the exact path the tree took:
# Glucose=138 > 127.5? YES →
# BMI=33.6 > 29.95?    YES → ⚠ Diabetes
print("Tree's reasoning:")
print("  Q1: Is Glucose > 127.5?  138 > 127.5 → YES")
print("  Q2: Is BMI > 29.95?      33.6 > 29.95 → YES")

What You Built — Decision Tree vs Logistic Regression

No Scaling Needed

Trees ask "above or below X?" — they don't care if numbers are big or small

Gini Impurity

How the tree picks each question — lowest Gini = cleanest split = best question

Overfitting Risk

Unlimited tree → 100% train, 69% test. Always prune with max_depth

max_depth Pruning

Limits questions to 4 deep → 80% train, 77% test. Small gap = generalizes well

Feature Importance

Glucose (42%) matters most. SkinThickness (0%) ignored completely

Full Transparency

You can read exactly which questions led to any prediction — unlike logistic regression
