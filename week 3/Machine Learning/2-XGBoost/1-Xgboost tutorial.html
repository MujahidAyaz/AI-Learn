Step 3 of 9
4
Train the Model
Plain English We set the key parameters. The most important ones are n_estimators (how many rounds), learning_rate (how much each tree corrects), and max_depth (how deep each tree goes). XGBoost trees are intentionally shallow — usually just 3–6 levels deep.
python
# Key parameters:
#
# n_estimators=200   → run 200 boosting rounds (200 trees)
#
# learning_rate=0.1  → each tree only corrects 10% of the error
#                      small = more stable, needs more trees
#                      large = faster but can overshoot
#
# max_depth=4        → each tree is shallow (3-6 is typical)
#                      XGBoost trees are SMALL on purpose
#                      many shallow trees > few deep trees
#
# subsample=0.8      → each tree uses 80% of training rows
#                      (same bagging idea as Random Forest)
#
# colsample_bytree=0.8 → each tree uses 80% of features
#                        (same as max_features in Random Forest)
#
# use_label_encoder=False → suppress a deprecation warning
# eval_metric="logloss"   → use log loss to measure errors

model = XGBClassifier(
    n_estimators=200,
    learning_rate=0.1,
    max_depth=4,
    subsample=0.8,
    colsample_bytree=0.8,
    use_label_encoder=False,
    eval_metric="logloss",
    random_state=42
)

model.fit(X_train, y_train)
print("XGBoost training complete!")
print(f"Trees built : {model.n_estimators}")
Output
XGBoost training complete!
Trees built : 200
Rule of thumb Lower learning_rate + more n_estimators = better accuracy but slower. A common combo: learning_rate=0.05, n_estimators=500. Start with learning_rate=0.1, n_estimators=200 and tune from there.

Early Stopping — Don't Overtrain
Plain English With 200 trees, how do we know when to stop? More trees isn't always better — at some point the model starts memorizing noise again. Early stopping watches the validation score and stops automatically when it stops improving.
python
# Early stopping — stop when validation score stops improving
# early_stopping_rounds=20 → stop if no improvement for 20 rounds
# eval_set → a validation set to monitor during training

model_es = XGBClassifier(
    n_estimators=500,        # set high — early stopping will cut it short
    learning_rate=0.05,
    max_depth=4,
    subsample=0.8,
    colsample_bytree=0.8,
    use_label_encoder=False,
    eval_metric="logloss",
    early_stopping_rounds=20,  # stop if no improvement for 20 rounds
    random_state=42
)

model_es.fit(
    X_train, y_train,
    eval_set=[(X_test, y_test)],   # monitor test set performance
    verbose=50                    # print score every 50 rounds
)

print(f"\nBest round    : {model_es.best_iteration}")
print(f"Best val loss : {model_es.best_score:.4f}")
print("Early stopping found the optimal number of trees automatically!")

Feature Importance
Plain English XGBoost gives you three different ways to measure feature importance — each tells you something slightly different. We'll look at all three so you understand which one to use when.
python
# XGBoost has 3 types of feature importance:
#
# "weight"   → how many times a feature was used to split
# "gain"     → average improvement in accuracy when feature splits
#              (most useful — tells you how GOOD each split is)
# "cover"    → how many patients a feature's splits affect

for importance_type in ["weight", "gain", "cover"]:
    scores = model_es.get_booster().get_score(
        importance_type=importance_type
    )
    # Sort by importance
    sorted_scores = sorted(scores.items(),
                            key=lambda x: x[1], reverse=True)
    print(f"\nImportance type: {importance_type.upper()}")
    for feat, score in sorted_scores:
        bar = "█" * int(score / max(scores.values()) * 30)
        print(f"  {feat:<30} {score:>8.1f}  {bar}")

Visualizations
Plain English Three charts: ROC curve, feature importance by gain, and confusion matrix. Plus the learning curve showing how the loss dropped round by round during training.
python
# Get training history (loss per round)
results  = model_es.evals_result()
val_loss = results["validation_0"]["logloss"]

cm  = confusion_matrix(y_test, y_pred)
fpr, tpr, _ = roc_curve(y_test, y_proba)

# Feature importance by gain
gain_scores = model_es.get_booster().get_score(importance_type="gain")
feat_names  = list(gain_scores.keys())
feat_vals   = list(gain_scores.values())
sorted_idx  = np.argsort(feat_vals)

fig, axes = plt.subplots(1, 4, figsize=(20, 5))
fig.suptitle("XGBoost — Diabetes Prediction",
             fontsize=14, fontweight="bold")

# Chart 1: Training loss curve
axes[0].plot(val_loss, color="#f59e0b", lw=2)
axes[0].axvline(model_es.best_iteration, color="#fb7185",
               linestyle="--", label=f"Best round={model_es.best_iteration}")
axes[0].set_title("Loss Per Round\n(early stopping)")
axes[0].set_xlabel("Round"); axes[0].set_ylabel("Log Loss")
axes[0].legend()

# Chart 2: ROC Curve
axes[1].plot(fpr, tpr, color="#f59e0b", lw=2, label=f"AUC={auc:.3f}")
axes[1].plot([0,1],[0,1],"--",color="#555")
axes[1].set_title("ROC Curve"); axes[1].legend()

# Chart 3: Feature Importance (gain)
axes[2].barh(np.array(feat_names)[sorted_idx],
             np.array(feat_vals)[sorted_idx], color="#f59e0b")
axes[2].set_title("Feature Importance\n(by gain)")

# Chart 4: Confusion Matrix
axes[3].imshow(cm, cmap="YlOrBr")
axes[3].set_xticks([0,1]); axes[3].set_yticks([0,1])
axes[3].set_xticklabels(["Pred: No","Pred: Yes"])
axes[3].set_yticklabels(["Actual: No","Actual: Yes"])
axes[3].set_title("Confusion Matrix")
for i in range(2):
    for j in range(2):
        axes[3].text(j, i, str(cm[i,j]), ha="center", va="center",
                     fontsize=16, fontweight="bold",
                     color="white" if cm[i,j]>cm.max()/2 else "black")

plt.tight_layout()
