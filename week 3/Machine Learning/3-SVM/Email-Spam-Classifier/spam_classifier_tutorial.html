Stage 1 — Problem Understanding
1
What Are We Actually Solving?
Plain English Before writing a single line of model code, we get precise about the problem. Inboxes get flooded with spam — scams, phishing, ads. We want a system that reads an incoming message and decides: spam, or not spam?
problem framing
# ML framing
Task   = Binary text classification
Input  = raw message text
Output = spam (1)  or  ham (0)

# What does success actually mean here?
#
# RECALL on spam   -> catch as much spam as possible
#                     (missing spam = it reaches the inbox)
#
# PRECISION on spam -> don't flag real messages as spam
#                      (a missed invite = broken trust)
#
# These two goals fight each other. We need a balance (F1)
# and we MUST inspect this tradeoff explicitly later.
Why this matters before any code Spam is the minority class in real life — usually 10-15% of messages. A model that lazily predicts "ham" every single time would already score ~87% accuracy while being completely useless. Defining success correctly up front prevents us from being fooled by a misleading metric later.
binary classification precision vs recall tradeoff class imbalance

Step 4 of 12
Stage 5 — Preprocessing
5
Normalize the Text
Plain English "FREE!!!" and "free" mean the same thing to a human but look completely different to a computer. We standardize every message the same way so the model sees real patterns, not formatting noise.
python
import re

def clean_text(text):
    text = text.lower()
    text = re.sub(r"http\S+|www\S+", " URL ", text)
    text = re.sub(r"\b\d{5,}\b", " PHONENUM ", text)
    text = re.sub(r"[^a-z\s]", " ", text)
    text = re.sub(r"\s+", " ", text).strip()
    return text

df["clean_message"] = df["message"].apply(clean_text)
print(df.iloc[2]["message"])
print(df.iloc[2]["clean_message"])

Step 5 of 12
Real Bug Fix — Found While Building This
⚠
The Empty String Bug
This actually happened while building this project Messages that are ONLY emoji or symbols — like ":)" — become an empty string after stripping everything non-alphabetic. An empty string crashes TF-IDF, and silently becomes NaN when the cleaned data is saved and reloaded from CSV.
the crash
ValueError: np.nan is an invalid document,
expected byte or unicode string.
python — the fix
def clean_text(text):
    text = text.lower()
    text = re.sub(r"http\S+|www\S+", " URL ", text)
    text = re.sub(r"\b\d{5,}\b", " PHONENUM ", text)
    text = re.sub(r"[^a-z\s]", " ", text)
    text = re.sub(r"\s+", " ", text).strip()

    # Messages that are ONLY emoticons/numbers/symbols
    # (e.g. ":)" or "645") become empty after cleaning.
    # An empty string breaks TF-IDF and gets misread as
    # NaN when saved/reloaded from CSV. Replace with a
    # safe placeholder instead of silently losing the row.
    if text == "":
        text = "emptymsg"

    return text
Why this is worth showing you Real datasets always have edge cases like this. Three messages in this exact dataset were just :) or numbers. The fix isn't complicated, but finding it required actually running the pipeline and reading the error — not just writing code that looks right.
real bug edge case handling fixed and documented

Stage 7 — Train/Test Split + TF-IDF
7
Split Fairly, Then Vectorize
Plain English We split 80/20 like always, but this time with stratify=y — it guarantees both sets keep the same 87/13 ratio. Then TF-IDF turns each message into a row of numbers, one column per word, weighted by how informative that word is.
python
from sklearn.model_selection import train_test_split
from sklearn.feature_extraction.text import TfidfVectorizer

X_train_text, X_test_text, y_train, y_test = train_test_split(
    df["clean_message"], df["label_num"],
    test_size=0.2, random_state=42,
    stratify=df["label_num"]   # keeps 87/13 ratio in both sets
)

# ngram_range=(1,2) -> captures single words AND 2-word phrases
# max_features=3000 -> keep only the 3000 most useful word columns
tfidf = TfidfVectorizer(
    max_features=3000, ngram_range=(1,2),
    min_df=2, stop_words="english"
)

X_train_tfidf = tfidf.fit_transform(X_train_text)
X_test_tfidf  = tfidf.transform(X_test_text)  # transform only — no relearning

print(f"Train spam ratio: {y_train.mean():.1%}")
print(f"Test spam ratio:  {y_test.mean():.1%}")
print(f"TF-IDF shape: {X_train_tfidf.shape}")
Output

Step 8 of 12
Stage 8 — Model Training + Hyperparameter Tuning
8
SVM with GridSearchCV
Plain English We use SVM because TF-IDF text produces high-dimensional, sparse data — exactly what SVM handles best. class_weight="balanced" auto-corrects for the 87/13 imbalance. We don't guess the C parameter — GridSearchCV tests 5 values and picks the best one using cross-validated F1.
python
from sklearn.svm import LinearSVC
from sklearn.model_selection import GridSearchCV

param_grid = {"C": [0.01, 0.1, 1, 10, 100]}

grid_search = GridSearchCV(
    LinearSVC(class_weight="balanced", max_iter=5000, random_state=42),
    param_grid, cv=5, scoring="f1", n_jobs=-1
)
grid_search.fit(X_train_tfidf, y_train)

print(f"Best C: {grid_search.best_params_}")
print(f"Best CV F1: {grid_search.best_score_:.4f}")

The classic overfitting curve Notice F1 rises then falls as C increases. C=1 is the sweet spot — too low and the margin is too soft to separate classes well, too high and the model overfits to training noise. Same tradeoff you've seen with max_depth and K, just a different knob.

Save Once, Predict Forever
Plain English Training takes time. We don't want to retrain every time we want a prediction. joblib saves the trained model and the TF-IDF vectorizer to disk — load them instantly later, no retraining needed.

Critical: save BOTH files together The vectorizer is what turns new raw text into the exact numeric format the model was trained on. Save the model without the matching vectorizer, and every future prediction will silently break or be wrong — there's no error, just garbage results.

Step 11 of 12
Stage 11 — Prediction Pipeline
11
Production-Ready Predictions
Plain English This is the file that actually gets used — it loads the saved model and classifies brand new messages instantly, no retraining. This is what would sit behind an API, an email server filter, or a Chrome extension.
python
model = joblib.load("models/svm_spam_model.pkl")
vectorizer = joblib.load("models/tfidf_vectorizer.pkl")

def predict_message(message):
    cleaned = clean_text(message)  # SAME cleaning as training!
    vec = vectorizer.transform([cleaned])
    pred = model.predict(vec)[0]
    prob = model.predict_proba(vec)[0][1]
    return ("SPAM" if pred==1 else "HAM"), prob

# Test on brand new messages the model never saw
test_messages = [
    "Congratulations! You've won a $1000 gift card. Click here!",
    "Hey, are we still meeting for lunch tomorrow?",
    "URGENT: Your account suspended. Verify immediately.",
]

for msg in test_messages:
    label, prob = predict_message(msg)
    print(f"{label} ({prob:.1%}) -> {msg[:50]}")
